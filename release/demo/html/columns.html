Column Definition

One of the most common scenarios is to provide the data binding yourself, stopping the grid from auto discovering the columns. This is achieved by setting up the field-name attribute on the <TH> elements that you'll have to include in the markup of a single <TR> element, part of <THEAD>.
This alwo works for nested objects (e.g. field-name="address.postCode").

Once the field-name was set, a range of other options can tweak the behavior and appearance of the column. You can

  • Provide the display name for the header (display-name)
  • Turn sorting on/off (enable-sorting)
  • Turn column filtering on/off (enable-filtering)
  • Set the width or height of the column cells, in any measurement unit you want (cell-width/cell-height)
  • Specify the alignment for the data cells (display-align="center|left|right")
  • Specify the display format (display-format)
    • By default the format is interpreted as an angular filter (e.g. display-format="number:2")
    • You may also specify a method to be called (e.g display-format=".toShortDateTime()")
  • Don't forget the field-name attribute. It's the minimum requirement for column customization.
Custom Columns

Extra columns can be added in the body, even without defining the header. Tapping into the currently bound row item can be done through the gridItem scope property. More examples that are covering this topic are available in the "Templating" section.
