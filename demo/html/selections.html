Selections

Selections are enabled by default. They are controlled by the selection-mode attribute, which can be set verbatim to one of the following values, or alternatively via a one-way variable (e.g. selection-mode="{{mySelectionMode}}")

  • None
    Turns off item selections.
  • SingleRow
    Only one item can be selected.
  • MultiRow (default)
    Multiple items can be selected or de-selected, with either a mouse click or a tap on the item row. These selections are "sticky", meaning that de-selecting each item must be done either through user action or via script on the data-bound collection (e.g. mySelectedItems.splice(0)).
  • MultiRowWithKeyModifiers
    Multiple items can be selected or de-selected. The selection is "non-sticky", and takes the modifier keys CTRL and SHIFT into account:
    • When CTRL is pressed whilst clicking on a row, the item will get selected or de-selected, without intefering with the rest of the selection set.
    • When SHIFT is pressed whilst clicking on a row, the range of items starting from the last selected item to the one clicked will get selected, without intefering with the rest of the selection set. De-selections are not possible with this key modifier.
    • Having none of these key modifiers pressed, the selection set will be cleared.
Something else worth mentioning here is that the selection set persists across page changes.

The enable-selections and the enable-multi-row-selections attributes are deprecated and will be removed in a future version.
